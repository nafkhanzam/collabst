Here are the Guidelines to follow if you want to contribute to the Collabst Project.

Please do also have a look at the [Collabst Code of Conduct](/CODE_OF_CONDUCT.md), and act respectfully when interacting with others.

## Reporting a Bug 🪳 / Making a Feature Request💡

Before making an Issue, always check in the existing [Issues](https://github.com/collabst/collabst/issues) if the same request has not already been made. If an issue already exists, but you have additional feedback to add, just make a comment instead of creating a new issue.

## Submitting a Bug Fix / Feature Implementation🪛

### 1. Use of Coding Agents 🤖

You can use Coding Agents to help you contribute, however keep the following points in mind:
- **YOU are responsible for the code you submit**, independently of how it was written.
- It is not the job of the maintainers to test your code.
- If your code breaks partially or entirely the application, it is not the maintainers job to fix your changes, and the submission won't be accepted. 

### 2. Making a Pull Request

Whether submitting a Bug Fix or a New Feature, **KEEP THE CHANGES TO A MINIMAL: Make only one Fix/Addition per Pull Request**. If you want to do multiple things, split the list into multiple Pull Requests.
You can follow these steps:
- Identify a clear problem you want to solve (ONE bug fix or ONE feature addition)
- Fork the Collabst Repo.
- Make your code changes in your fork.
- Make a Pull Request: State clearly what the PR fixes/adds
- Eventually mention an associated Issue
- The maintainers will try there best to review the PR. Whether it is merged or not stays at their discretion.
- Don't spam them to reply and be respectful, remember you're talking to human beings.

<br/>
Have fun coding & thank you for your contributions !
